# User Feedback Collection Design Brief

## Overview

A sidebar-integrated feedback system that allows users to submit bug reports, feature requests, or praise directly from any page in the application. The design follows Linear-style minimalism with a clean modal interface, type selection via segmented control, and auth-aware form behavior.

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Trigger Location | Sidebar (not floating button) | Cleaner UI, integrates with existing navigation, less intrusive |
| Feedback Types | Bug, Feature, Praise | Covers primary feedback categories without overwhelming users |
| Type Selection | Segmented control with icons | Visual, quick selection, Linear-style minimal design |
| Email Field | Conditional (anonymous users only) | Reduces friction for logged-in users, optional for anonymous |
| Success Feedback | Toast notification | Non-blocking, confirms submission without modal |
| Modal Style | Dialog with fade animation | Consistent with existing UX patterns, accessible |

## Visual Direction

- **Theme:** Light mode primary (matches existing app theme)
- **Colors:** Slate palette for neutrals, semantic colors for feedback types:
  - Bug: Red accent (`text-red-600`)
  - Feature: Blue accent (`text-blue-600`)
  - Praise: Green accent (`text-green-600`)
- **Typography:** System font stack, consistent with app
- **Spacing:** Generous padding (p-6), clear visual hierarchy
- **Animations:** Subtle transitions (200ms), focus rings for accessibility
